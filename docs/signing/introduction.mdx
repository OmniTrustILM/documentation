# Introduction

ILM is bringing its digital-signing capabilities natively into the platform, built directly on ILM Core profiles, connectors, and supporting microservices. Capabilities are delivered incrementally; **timestamping** is the first to land.

During the transition, two implementations are documented side by side:

- **ILM-native signing** — this section. Built on Core profiles and connectors.
- **SignServer-based signing** — see [Digital Signing — legacy](/docs/signserver/introduction). The established implementation, supported during the transition.

## Timestamping

Timestamping is the first ILM-native signing capability. It issues both standard RFC 3161 tokens and **qualified electronic time stamps** under eIDAS (Regulation (EU) No 910/2014) Art. 42. Start with the [Timestamping overview](/docs/signing/timestamping/overview), then explore the rest of the section:
